# SEO Rulebook for State Service Pages (Google SEO + AEO + LLM)

Last Updated: 2026-05-23  
Project: IntegraFin (`integrafin.tax`)  
Target Markets: Texas, New York, Pennsylvania

## 1) Purpose

This file defines the exact content and SEO rules for all new state and city pages so they rank better in Google (SEO), appear in answer engines (AEO), and are easier for AI/LLM systems to cite.

## 2) Core Google Rules We Must Follow

1. Write for people first, not search engines first.
2. Demonstrate experience and trust for finance/tax topics (YMYL).
3. Keep content accurate, updated, and state-specific.
4. Avoid thin pages and location doorway pages.
5. Use one canonical URL per page intent.
6. Make every page crawlable, indexable, and internally linked.
7. Keep spam signals out: keyword stuffing, copied content, hidden text, fake reviews.

## 3) Page Intent Rules (State and City)

1. One page = one main intent.
2. State page intent: broad service discovery for that state.
3. City page intent: local commercial intent for that city.
4. Blog page intent: informational support that feeds service pages.
5. Do not mix multiple unrelated intents on one page.

## 4) Mandatory On-Page Structure for State Pages

Every state page must include these sections in this order:

1. Hero with clear H1 including service + state.
2. Direct answer paragraph (2 to 4 lines) explaining who we help.
3. Services list for that state.
4. Who we help (business types, individual types).
5. State-specific compliance/tax context.
6. Service areas/city mentions in that state.
7. FAQ (5 to 8 real questions).
8. Trust block (credentials, process, response time, contact).
9. Strong CTA (book call, request callback).

## 5) Human-Style Content Rules (Non-AI Slop)

1. Use plain English and short sentences.
2. Add real scenarios (example: "Texas LLC with quarterly sales tax").
3. Add practical steps, not generic claims.
4. Avoid repeated phrases across pages.
5. Keep each state page uniquely written (minimum 60% unique body text).
6. Use active voice and direct guidance.
7. Remove fluff and exaggerated promises.

## 6) Keyword Optimization Rules

1. One primary keyword per page.
2. 4 to 8 secondary/support keywords naturally in headings/body.
3. Primary keyword placement:
- Title tag
- H1
- First 120 words
- One H2
- Meta description
- URL slug
4. Use semantic variants, not exact-match repetition.
5. Keep keyword density natural; never force it.

## 8) Title, Meta, URL Rules

1. Title length: 50 to 60 characters.
2. Meta description: 140 to 160 characters with value + CTA.
3. URL must be short and intent-based.
4. Use canonical tag for each page.
5. Use only one H1.

## 9) Internal Linking Rules

1. Every state page must link to:
- `/services`
- `/contact`
- at least 2 relevant blog posts
2. Homepage must link to all state hub pages.
3. Services page must link to all state hub pages.
4. Blog posts must link back to one service page and one state page.
5. Use descriptive anchor text, not "click here".

## 10) Schema Rules

Required schema types:

1. `Organization`
2. `LocalBusiness`
3. `Service` (on service-focused pages)
4. `FAQPage` (when FAQ visible on page)
5. `BreadcrumbList`

Schema quality rules:

1. Schema must match visible page content.
2. No fake ratings/reviews schema.
3. Keep business NAP details consistent.

## 11) AEO (Answer Engine Optimization) Rules

1. Add a short direct answer under every major heading.
2. Use Q/A blocks for high-intent queries.
3. Keep answers factual and concise first, detail second.
4. Use bullet steps for processes.
5. Add definitions for tax terms in simple language.

## 12) LLM and AI Search Optimization Rules

1. Publish citation-friendly content (facts, checklists, clear claims).
2. Include "Last reviewed" date on YMYL pages.
3. Add expert attribution (CPA/EA/team reviewer).
4. Use clear section titles and consistent hierarchy.
5. Keep policy, contact, and trust pages updated.
6. Avoid unverifiable claims like "best in USA" without evidence.

## 13) E-E-A-T Trust Rules for Tax Content

1. Show credentialed reviewer/author where possible.
2. Include business address, phone, and email.
3. Include privacy and terms links.
4. Show service process and expected turnaround.
5. Add real proof points (years, industries served, common case types).

## 14) Technical Rules Per Page Launch

1. Confirm status code is 200.
2. Confirm indexable (no `noindex`).
3. Confirm canonical is correct.
4. Confirm page appears in sitemap.
5. Confirm internal links are present.
6. Confirm mobile rendering and speed pass.
7. Submit URL in Search Console for indexing.

## 15) Content Quality Gate (Publish/No Publish)

Page is publish-ready only if all are true:

1. Unique content with state-specific context.
2. Primary keyword aligned with intent.
3. Trust signals included.
4. FAQ section added.
5. Internal links added.
6. Metadata and canonical finalized.
7. Grammar and readability reviewed by human.

## 16) Forbidden Practices

1. Copy-paste same text across TX/NY/PA pages.
2. Auto-generated filler paragraphs without local value.
3. Keyword stuffing in headings.
4. Creating dozens of thin city pages without unique value.
5. Fake local office claims for cities with no presence.

## 17) Weekly Execution Standard

1. Publish 2 quality pages/posts.
2. Update 2 older pages for freshness.
3. Add 10 internal links across relevant pages.
4. Review Search Console for indexing/ranking movement.
5. Track leads from organic and top landing pages.

## 18) State Page Writing Template (Use Every Time)

1. H1: `[State] Tax and Accounting Services for Businesses and Individuals`
2. Intro: who, what, where, CTA.
3. H2: `Tax and Accounting Services in [State]`
4. H2: `Who We Help in [State]`
5. H2: `Why Businesses Choose IntegraFin in [State]`
6. H2: `Frequently Asked Questions`
7. Final CTA: `Book a Consultation`

## 19) Quick Compliance Checklist

- [ ] Title + meta optimized
- [ ] H1 and first paragraph include primary keyword naturally
- [ ] State-specific content included
- [ ] FAQ section included
- [ ] Service + Contact internal links included
- [ ] Canonical and sitemap verified
- [ ] Schema valid and matches page content
- [ ] URL submitted in Search Console
